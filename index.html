Spring boot配置pom打jar包--方式一

发表于 2018-12-11 更新于 2020-03-05 分类于 Spring Boot

方式一、使用maven-compiler-plugin插件指定java编译的版本

<build>
<!-- 打成jar包的名称-->
<finalName>cyam-lzh</finalName>
<plugins>
        <!-- Spring Boot打包插件 -->
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>    
        <!-- 指定项目源码的jdk版本，编译后的jdk版本，以及编码 -->
        <plugin>  
            <groupId>org.apache.maven.plugins</groupId>  
            <artifactId>maven-compiler-plugin</artifactId>  
            <configuration>
                <source>1.8</source>  
                <target>1.8</target>  
                <archive>
                    <manifest>
                        <addClasspath>true</addClasspath>
                        <classpathPrefix>lib</classpathPrefix>
                        <!-- SpringBoot 启动的类的路径-->
                        <mainClass>com.boot.Application</mainClass>
                    </manifest>
                </archive>
            </configuration>  
        </plugin>
    </plugins>
    <resources>
        <!--指定将src/main/resources/资源目录下的所有文件都打入 -->
        <resource>
            <directory>src/main/resources/</directory>
            <includes>
                <include>**/*.*</include>
            </includes>
        </resource>
    </resources>
  </build>

Spring boot配置pom打jar包--方式二

发表于 2018-12-11 更新于 2020-03-05 分类于 Spring Boot

方式二的打包方式和方式一的打包方式基本无恙，主要差别是指定了maven-assembly-plugin插件定制化打包

<build>
<finalName>cyam-lzh</finalName>
<plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>    
        <!-- 使用该插件仅仅指定编译java时的版本-->
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>  
            <artifactId>maven-compiler-plugin</artifactId>  
            <configuration> 
                <!-- 指定编译时使用的jdk版本--> 
                <source>1.8</source>  
                <target>1.8</target>  
            </configuration>  
        </plugin>
        <!-- 使用maven-assembly-plugin插件定制化打包-->
        <plugin>
            <artifactId>maven-assembly-plugin</artifactId>
            <configuration>
                <archive>
                    <manifest>
                        <mainClass>com.boot.Application</mainClass>
                    </manifest>
                </archive>
                <descriptorRefs>
                    <descriptorRef>
                        jar-with-dependencies
                    </descriptorRef>
                </descriptorRefs>
            </configuration>
        </plugin>    
    </plugins>
    <resources>
        <!--指定将src/main/resources/资源目录下的所有文件都打入 -->
        <resource>
            <directory>src/main/resources/</directory>
            <includes>
                <include>**/*.*</include>
            </includes>
        </resource>
    </resources>
  </build>

Mysql导入数据报1067的错误

发表于 2018-11-21 更新于 2020-03-05 分类于 MySQL

Error Code: 1067. Invalid default value for ‘createDate’
现有从mysql5.6+备份的数据库一份。需要导入到低版本5.5-中，然后报以上错误，
根据提示可知是createDate字段错误
错误原因：
因为CURRENT_TIMESTAMP函数在 5.6+ 版本才能赋值 DEFAULT 到 DATETIME 类型的数据中。
导入到低版本中由于DATETIME类型不支持CURRENT_TIMESTAMP函数，因此报错。
解决办法：
将字段’createDate’的类型改为timestamp即可解决。

Maven 引入本地jar包的几种方式

发表于 2018-11-12 更新于 2020-03-05 分类于 Maven

方式一、通过将本地jar加入到Maven Repository中,命令如下 

mvn install:install-file -Dfile=本地jar包路径 -DgroupId=groupId名称 -DartifactId=artifactId名称 -Dversion=版本号 -Dpackaging=jar

例如:
  我们现有一个百度的ueditor包，现在需要通过maven将其引入到项目中。具体的命令为： 

mvn install:install-file -Dfile=D:/jar/ueditor.jar-DgroupId=com.baidu -DartifactId=ueditor -Dversion=1.1.2 -Dpackaging=jar

执行上述命令之后（前提是你已经安装过了maven，并配置了环境变量），我们在pom文件中即可引入该jar包，如下

<dependency>
    <groupId>com.baidu</groupId>
    <artifactId>ueditor</artifactId>
    <version>1.1.2</version>
</dependency>

缺点： 如果是公用一个仓库还好，如果是单独的仓库，凡是加入该项目组的成员，都需要执行该命令，可移植性不好。
